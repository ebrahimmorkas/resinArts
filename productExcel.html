Key Points:

  • Multiple rows per product: Each row represents one size within a variant
  • Product without variants: Single row with all basic product info
  • JSON format: Use JSON strings for complex data like details and bulk pricing
  • Pipe separation: Use | to separate multiple images

Data Format Explanations

JSON Format Examples:

  • productDetails: {"key1":"value1","key2":"value2"}
  • bulkPricing: [{"wholesalePrice":20,"quantity":10},{"wholesalePrice":15,"quantity":20}]
  • sizeOptionalDetails: {"Material":"Cotton","Care":"Machine Wash"}
  • Empty arrays: [] for no data

Special Handling:

  • Multiple Images: Use pipe separator: image1.jpg|image2.jpg|image3.jpg
  • Category IDs: Use MongoDB ObjectId strings or category names (your choice)
  • Dates: Use ISO format: 2025-09-26T05:48:00.000Z
  • Boolean Values: Use TRUE/FALSE (Excel will convert to boolean)
  • Empty Fields: Leave blank for null/empty values
